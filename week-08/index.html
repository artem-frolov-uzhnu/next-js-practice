1. Підготовка проєкту

Завдання: Встановити залежності

Встановіть NextAuth.js та бібліотеку для хешування паролів:

Terminal (в папці проекту)
npm install next-auth bcryptjs
Що ми встановлюємо:
  • next-auth — бібліотека автентифікації для Next.js (підтримує OAuth, Credentials, JWT тощо)
  • bcryptjs — хешування паролів (JavaScript-реалізація bcrypt, не потребує компіляції C++)

Завдання: Налаштувати змінні середовища

Додайте нові змінні до файлу .env.local:

.env.local (додати до існуючого)
# ... існуючий MONGODB_URI ...

# NextAuth.js
NEXTAUTH_SECRET=ваш_секретний_ключ_тут
NEXTAUTH_URL=http://localhost:3000

Для генерації надійного секретного ключа виконайте команду в терміналі:

PowerShell
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])

Або якщо встановлено OpenSSL (наприклад, через Git Bash):

Git Bash
openssl rand -base64 32
Terminal
openssl rand -base64 32
Terminal
openssl rand -base64 32

Скопіюйте згенерований рядок і вставте замість ваш_секретний_ключ_тут.

Навіщо NEXTAUTH_SECRET? Цей ключ використовується для шифрування JWT-токенів та cookies сесії. Без нього NextAuth.js не зможе безпечно зберігати стан автентифікації. Ніколи не комітьте цей ключ у Git!
✅ Контрольна точка
  • next-auth та bcryptjs встановлено (перевірте в package.json)
  • .env.local містить NEXTAUTH_SECRET та NEXTAUTH_URL
  • NEXTAUTH_SECRET — це довгий випадковий рядок (не «ваш_секретний_ключ_тут»!)

2. Модель User

Модель User зберігатиме інформацію про зареєстрованих користувачів: ім’я, email, хешований пароль та роль (user або admin).

Завдання: Створити модель User

Створіть файл lib/models/User.js:

lib/models/User.js
import mongoose from 'mongoose'

const userSchema = new mongoose.Schema({
  name: {
    type: String,
    required: [true, "Ім'я обов'язкове"],
    trim: true,
    maxlength: [50, "Ім'я не може бути довшим за 50 символів"],
  },
  email: {
    type: String,
    required: [true, 'Email обов\'язковий'],
    unique: true,
    lowercase: true,
    trim: true,
    match: [/^\S+@\S+\.\S+$/, 'Невалідний формат email'],
  },
  password: {
    type: String,
    required: [true, 'Пароль обов\'язковий'],
    minlength: [6, 'Пароль має містити щонайменше 6 символів'],
    select: false,
  },
  role: {
    type: String,
    enum: {
      values: ['user', 'admin'],
      message: 'Роль має бути: user або admin',
    },
    default: 'user',
  },
}, {
  timestamps: true,
})

export default mongoose.models.User || mongoose.model('User', userSchema)
Ключові особливості моделі:
  • email: { unique: true } — MongoDB створить унікальний індекс, два користувачі з однаковим email неможливі
  • password: { select: false } — пароль не повертається у запитах за замовчуванням. Це важливо для безпеки: навіть якщо ви випадково повернете користувача в API, хеш пароля не потрапить у відповідь
  • role: enum ['user', 'admin'] — підготовка до RBAC (розмежування ролей), яке ми реалізуємо на наступному тижні
  • lowercase: true — email автоматично приводиться до нижнього регістру
✅ Контрольна точка
  • Файл lib/models/User.js створено
  • Схема має поля: name, email (unique), password (select: false), role (enum)
  • Є валідація з кастомними повідомленнями
  • Використовується timestamps: true

3. Налаштування NextAuth.js

NextAuth.js потребує конфігурації: який провайдер автентифікації використовувати, як створювати та перевіряти сесії, та які дані зберігати в токені.

Завдання: Створити конфігурацію NextAuth

Створіть файл lib/auth.js:

lib/auth.js
import CredentialsProvider from 'next-auth/providers/credentials'
import bcrypt from 'bcryptjs'
import dbConnect from '@/lib/db'
import User from '@/lib/models/User'

export const authOptions = {
  providers: [
    CredentialsProvider({
      name: 'credentials',
      credentials: {
        email: { label: 'Email', type: 'email' },
        password: { label: 'Пароль', type: 'password' },
      },
      async authorize(credentials) {
        if (!credentials?.email || !credentials?.password) {
          throw new Error('Введіть email та пароль')
        }

        await dbConnect()

        // Знаходимо користувача і явно запитуємо пароль
        const user = await User.findOne({
          email: credentials.email
        }).select('+password')

        if (!user) {
          throw new Error('Невірний email або пароль')
        }

        // Порівнюємо паролі
        const isPasswordCorrect = await bcrypt.compare(
          credentials.password,
          user.password
        )

        if (!isPasswordCorrect) {
          throw new Error('Невірний email або пароль')
        }

        // Повертаємо об'єкт користувача (без пароля)
        return {
          id: user._id.toString(),
          name: user.name,
          email: user.email,
          role: user.role,
        }
      },
    }),
  ],

  session: {
    strategy: 'jwt',
  },

  callbacks: {
    async jwt({ token, user }) {
      // При першому вході додаємо дані в токен
      if (user) {
        token.role = user.role
        token.id = user.id
      }
      return token
    },

    async session({ session, token }) {
      // Додаємо дані з токена в сесію
      if (session.user) {
        session.user.role = token.role
        session.user.id = token.id
      }
      return session
    },
  },

  pages: {
    signIn: '/auth/login',
  },

  secret: process.env.NEXTAUTH_SECRET,
}
Як працює CredentialsProvider:
  1. Користувач вводить email та пароль на сторінці входу
  2. NextAuth викликає функцію authorize() з цими даними
  3. Ми шукаємо користувача в MongoDB та перевіряємо пароль через bcrypt.compare()
  4. Якщо все вірно — повертаємо об’єкт користувача (NextAuth створить JWT-токен)
  5. Якщо ні — кидаємо помилку (NextAuth покаже повідомлення)
Навіщо callbacks? За замовчуванням JWT-токен та сесія містять тільки name, email, image. Щоб додати role та id, ми використовуємо callbacks:
  • jwt — додає дані в токен при його створенні
  • session — копіює дані з токена в об’єкт сесії, доступний на клієнті

Завдання: Створити API Route для NextAuth

Створіть файл app/api/auth/[...nextauth]/route.js:

app/api/auth/[...nextauth]/route.js

import { authOptions } from '@/lib/auth'

const handler = NextAuth(authOptions)

export { handler as GET, handler as POST }

Цей catch-all route обробляє всі запити NextAuth: /api/auth/signin, /api/auth/signout, /api/auth/session тощо.

Завдання: Створити AuthProvider

NextAuth потребує SessionProvider для роботи хука useSession() на клієнті. Створіть файл components/AuthProvider.jsx:

components/AuthProvider.jsx
'use client'

import { SessionProvider } from 'next-auth/react'

export default function AuthProvider({ children }) {
  return (

      {children}

Чому окремий компонент? SessionProvider — це Client Component (використовує React Context). Файл layout.js за замовчуванням — Server Component. Щоб не робити весь layout клієнтським, ми обгортаємо тільки SessionProvider в окремий Client Component.

4. API реєстрації

NextAuth.js не має вбудованого механізму реєстрації — тільки вхід. Ми створимо окремий API-ендпоінт, який прийматиме дані нового користувача, валідуватиме їх, хешуватиме пароль і зберігатиме в базі.

Завдання: Створити API endpoint реєстрації

Створіть файл app/api/auth/register/route.js:

app/api/auth/register/route.js
import bcrypt from 'bcryptjs'
import dbConnect from '@/lib/db'
import User from '@/lib/models/User'

export async function POST(request) {
  try {
    const { name, email, password } = await request.json()

    // Валідація
    if (!name || !email || !password) {
      return Response.json(
        { error: "Всі поля обов'язкові" },
        { status: 400 }
      )
    }

    if (password.length < 6) {
      return Response.json(
        { error: 'Пароль має містити щонайменше 6 символів' },
        { status: 400 }
      )
    }

    await dbConnect()

    // Перевірка чи email вже зайнятий
    const existingUser = await User.findOne({ email })

    if (existingUser) {
      return Response.json(
        { error: 'Користувач з таким email вже існує' },
        { status: 409 }
      )
    }

    // Хешуємо пароль
    const hashedPassword = await bcrypt.hash(password, 12)

    // Створюємо користувача
    const user = await User.create({
      name,
      email,
      password: hashedPassword,
    })

    return Response.json(
      {
        message: 'Реєстрація успішна',
        user: {
          id: user._id,
          name: user.name,
          email: user.email,
          role: user.role,
        },
      },
      { status: 201 }
    )
  } catch (error) {
    if (error.name === 'ValidationError') {
      const messages = Object.values(error.errors)
        .map((err) => err.message)
      return Response.json(
        { error: messages.join(', ') },
        { status: 400 }
      )
    }

    return Response.json(
      { error: 'Помилка сервера' },
      { status: 500 }
    )
  }
}
Як працює хешування паролів:
  • bcrypt.hash(password, 12) — хешує пароль з «сіллю» (12 раундів). Кожен раз генерується унікальний хеш, навіть для однакових паролів
  • bcrypt.compare(password, hash) — перевіряє, чи пароль відповідає хешу. Використовується в authorize() при вході
  • Ми ніколи не зберігаємо пароль у відкритому вигляді — тільки хеш
✅ Контрольна точка

Протестуйте реєстрацію через DevTools Console (F12):

⚠️ Безпека: чому 409 а не загальна помилка?

У навчальному проекті ми повертаємо конкретне повідомлення «email вже існує» для зручності. У реальних production-додатках це може бути вразливістю (атакуючий може перевіряти, чи зареєстрований певний email). Тоді використовують загальне повідомлення: «Не вдалося створити акаунт».

5. Сторінки входу та реєстрації

Використаємо Route Group (auth) для сторінок автентифікації. Route Group дозволяє організувати файли в папку без впливу на URL: app/(auth)/auth/login/page.js матиме URL /auth/login.

Завдання: Створити компонент LoginForm

Створіть файл components/LoginForm.jsx:

components/LoginForm.jsx
'use client'

import { useState } from 'react'
import { signIn } from 'next-auth/react'
import { useRouter } from 'next/navigation'
import Link from 'next/link'

export default function LoginForm() {
  const router = useRouter()
  const [formData, setFormData] = useState({
    email: '',
    password: '',
  })
  const [error, setError] = useState(null)
  const [isLoading, setIsLoading] = useState(false)

  const handleChange = (e) => {
    setFormData({ ...formData, [e.target.name]: e.target.value })
  }

  const handleSubmit = async (e) => {
    e.preventDefault()
    setError(null)
    setIsLoading(true)

    try {
      const result = await signIn('credentials', {
        email: formData.email,
        password: formData.password,
        redirect: false,
      })

      if (result?.error) {
        setError('Невірний email або пароль')
        setIsLoading(false)
        return
      }

      router.push('/dashboard')
      router.refresh()
    } catch {
      setError('Щось пішло не так')
      setIsLoading(false)
    }
  }

  return (
    <div className="max-w-md mx-auto mt-10">
      <div className="bg-white rounded-lg shadow p-8">
        <h1 className="text-2xl font-bold text-center mb-6">
          Вхід
        </h1>

        {error && (
          <div className="bg-red-50 border border-red-200
            text-red-700 px-4 py-3 rounded mb-4">
            {error}
          </div>
        )}

        <form onSubmit={handleSubmit} className="space-y-4">
          <div>
            <label className="block text-gray-700
              font-bold mb-2">
              Email
            </label>
            <input
              type="email"
              name="email"
              value={formData.email}
              onChange={handleChange}
              required
              className="w-full px-4 py-2 border rounded
                focus:outline-none focus:border-amber-500"
            />
          </div>

          <div>
            <label className="block text-gray-700
              font-bold mb-2">
              Пароль
            </label>
            <input
              type="password"
              name="password"
              value={formData.password}
              onChange={handleChange}
              required
              className="w-full px-4 py-2 border rounded
                focus:outline-none focus:border-amber-500"
            />
          </div>

          <button
            type="submit"
            disabled={isLoading}
            className="w-full bg-amber-700 text-white py-2
              rounded hover:bg-amber-800 font-bold
              disabled:opacity-50"
          >
            {isLoading ? 'Вхід...' : 'Увійти'}
          </button>
        </form>

        <p className="text-center mt-4 text-gray-600">
          Немає акаунту?{' '}
          <Link href="/auth/register"
            className="text-amber-700 hover:underline">
            Зареєструватися
          </Link>
        </p>
      </div>
    </div>
  )
}
Як працює signIn('credentials', { redirect: false }):
  • signIn — функція NextAuth, яка ініціює процес входу
  • 'credentials' — ім’я провайдера (ми вказали його в lib/auth.js)
  • redirect: false — не перенаправляти автоматично, ми обробимо результат самі
  • Якщо result.error — вхід невдалий, показуємо повідомлення
  • Якщо вхід успішний — перенаправляємо на /dashboard і оновлюємо сторінку

Завдання: Створити компонент RegisterForm

Створіть файл components/RegisterForm.jsx:

components/RegisterForm.jsx
'use client'

import { useState } from 'react'
import { signIn } from 'next-auth/react'
import { useRouter } from 'next/navigation'
import Link from 'next/link'

export default function RegisterForm() {
  const router = useRouter()
  const [formData, setFormData] = useState({
    name: '',
    email: '',
    password: '',
    confirmPassword: '',
  })
  const [error, setError] = useState(null)
  const [isLoading, setIsLoading] = useState(false)

  const handleChange = (e) => {
    setFormData({ ...formData, [e.target.name]: e.target.value })
  }

  const handleSubmit = async (e) => {
    e.preventDefault()
    setError(null)

    // Перевірка паролів
    if (formData.password !== formData.confirmPassword) {
      setError('Паролі не збігаються')
      return
    }

    setIsLoading(true)

    try {
      // 1. Реєстрація
      const res = await fetch('/api/auth/register', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          name: formData.name,
          email: formData.email,
          password: formData.password,
        }),
      })

      const data = await res.json()

      if (!res.ok) {
        setError(data.error || 'Помилка реєстрації')
        setIsLoading(false)
        return
      }

      // 2. Автоматичний вхід після реєстрації
      const result = await signIn('credentials', {
        email: formData.email,
        password: formData.password,
        redirect: false,
      })

      if (result?.error) {
        // Реєстрація пройшла, але вхід не вдався
        router.push('/auth/login')
        return
      }

      router.push('/dashboard')
      router.refresh()
    } catch {
      setError('Щось пішло не так')
      setIsLoading(false)
    }
  }

  return (
    <div className="max-w-md mx-auto mt-10">
      <div className="bg-white rounded-lg shadow p-8">
        <h1 className="text-2xl font-bold text-center mb-6">
          Реєстрація
        </h1>

        {error && (
          <div className="bg-red-50 border border-red-200
            text-red-700 px-4 py-3 rounded mb-4">
            {error}
          </div>
        )}

        <form onSubmit={handleSubmit} className="space-y-4">
          <div>
            <label className="block text-gray-700
              font-bold mb-2">
              Ім’я
            </label>
            <input
              type="text"
              name="name"
              value={formData.name}
              onChange={handleChange}
              required
              className="w-full px-4 py-2 border rounded
                focus:outline-none focus:border-amber-500"
            />
          </div>

          <div>
            <label className="block text-gray-700
              font-bold mb-2">
              Email
            </label>
            <input
              type="email"
              name="email"
              value={formData.email}
              onChange={handleChange}
              required
              className="w-full px-4 py-2 border rounded
                focus:outline-none focus:border-amber-500"
            />
          </div>

          <div>
            <label className="block text-gray-700
              font-bold mb-2">
              Пароль
            </label>
            <input
              type="password"
              name="password"
              value={formData.password}
              onChange={handleChange}
              required
              minLength={6}
              className="w-full px-4 py-2 border rounded
                focus:outline-none focus:border-amber-500"
            />
          </div>

          <div>
            <label className="block text-gray-700
              font-bold mb-2">
              Підтвердження пароля
            </label>
            <input
              type="password"
              name="confirmPassword"
              value={formData.confirmPassword}
              onChange={handleChange}
              required
              minLength={6}
              className="w-full px-4 py-2 border rounded
                focus:outline-none focus:border-amber-500"
            />
          </div>

          <button
            type="submit"
            disabled={isLoading}
            className="w-full bg-amber-700 text-white py-2
              rounded hover:bg-amber-800 font-bold
              disabled:opacity-50"
          >
            {isLoading ? 'Реєстрація...' : 'Зареєструватися'}
          </button>
        </form>

        <p className="text-center mt-4 text-gray-600">
          Вже маєте акаунт?{' '}
          <Link href="/auth/login"
            className="text-amber-700 hover:underline">
            Увійти
          </Link>
        </p>
      </div>
    </div>
  )
}

Структура Route Group (auth):
Файлова структура
app/
  (auth)/              ← Route Group (не впливає на URL)
    auth/
      login/
        page.js        ← URL: /auth/login
      register/
        page.js        ← URL: /auth/register

Route Group (auth) дозволяє в майбутньому додати окремий layout.js для сторінок автентифікації (наприклад, без Header та Footer).

✅ Контрольна точка

  • Увійдіть з тими ж даними — перенаправлення на /dashboard
  • Спробуйте невірний пароль — побачите повідомлення про помилку
⚠️ Типова помилка: «Cannot read properties of undefined (reading 'providers')»

Ця помилка означає, що NextAuth не може знайти конфігурацію. Перевірте: (1) файл app/api/auth/[...nextauth]/route.js створено в правильній директорії, (2) шлях імпорту @/lib/auth правильний, (3) authOptions експортується з lib/auth.js.

Завдання: Оновити Header.jsx

Оновіть components/Header.jsx:

components/Header.jsx (оновлений)
'use client'

import Link from 'next/link'
import { usePathname } from 'next/navigation'
import { useSession, signOut } from 'next-auth/react'

const navLinks = [
  { href: '/', label: 'Головна' },
  { href: '/menu', label: 'Меню' },
  { href: '/about', label: 'Про нас' },
  { href: '/contact', label: 'Контакти' },
]

export default function Header() {
  const pathname = usePathname()
  const { data: session, status } = useSession()

  return (
    <header className="bg-amber-900 text-white py-4">
      <div className="container mx-auto px-4
        flex justify-between items-center">
        <Link href="/"
          className="text-xl font-bold hover:text-amber-200
          transition">
          Кофеїн
        </Link>

        <nav className="flex items-center gap-6">
          <ul className="flex gap-6">
            {navLinks.map((link) => {
              const isActive =
                link.href === '/'
                  ? pathname === '/'
                  : pathname.startsWith(link.href)

              return (
                <li key={link.href}>
                  <Link href={link.href}
                    className={`transition ${
                      isActive
                        ? 'text-amber-300 font-semibold'
                        : 'hover:text-amber-200'
                    }`}>
                    {link.label}
                  </Link>
                </li>
              )
            })}
          </ul>

          {/* Блок автентифікації */}
          {status === 'loading' ? (
            <span className="text-amber-200 text-sm">...</span>
          ) : session ? (
            <div className="flex items-center gap-4">
              <Link href="/dashboard"
                className="text-amber-200 hover:text-white
                transition">
                {session.user.name}
              </Link>
              <button
                onClick={() => signOut({ callbackUrl: '/' })}
                className="bg-amber-700 hover:bg-amber-600
                  px-3 py-1 rounded text-sm transition"
              >
                Вийти
              </button>
            </div>
          ) : (
            <Link href="/auth/login"
              className="bg-amber-700 hover:bg-amber-600
              px-4 py-2 rounded transition">
              Увійти
            </Link>
          )}
        </nav>
      </div>
    </header>
  )
}
Як працює useSession():
  • status — 'loading', 'authenticated' або 'unauthenticated'
  • session — об’єкт з user: { name, email, role, id } (або null)
  • signOut({ callbackUrl: '/' }) — вихід з автоматичним перенаправленням на головну
  • Хук працює тільки всередині SessionProvider (ми додали його через AuthProvider)
✅ Контрольна точка
  • Коли не увійшли: в Header бачите кнопку «Увійти»
  • Коли увійшли: бачите ім’я користувача (клікабельне, веде на dashboard) та кнопку «Вийти»
  • Натисніть «Вийти» — повертає на головну, кнопка знову «Увійти»
  • Оновіть сторінку (F5) — стан сесії зберігається (якщо увійшли — лишаєтесь увійшовшим)
