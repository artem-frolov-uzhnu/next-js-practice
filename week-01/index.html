Приклад результату: На цьому занятті ви створите landing page для вашого майбутнього проекту. Викладач продемонструє приклад на темі "Кав'ярня", але ви маєте адаптувати його під свою обрану тему з 15 варіантів.

1. Підготовка середовища

Оберіть вашу операційну систему — інструкції відрізняються для різних ОС. Натисніть на вкладку з вашою системою.

1.1. Встановлення Node.js

Завдання: Встановити Node.js на Windows

  1. Відкрийте браузер та перейдіть на nodejs.org
  2. Завантажте версію LTS (Long Term Support) — зелена кнопка
  3. Запустіть завантажений файл node-vXX.X.X-x64.msi
  4. Слідуйте інструкціям інсталятора (Next → Next → Install)
  5. Перезапустіть комп'ютер (рекомендовано)

Перевірка встановлення:

PowerShell або Command Prompt (cmd)
# Відкрийте PowerShell (Win + X → Windows PowerShell)
# або Command Prompt (Win + R → cmd → Enter)

node -v
# Очікуваний результат: v20.x.x або v22.x.x

npm -v
# Очікуваний результат: 10.x.x
Якщо команда не знайдена ("not recognized")
  • Перезапустіть термінал після встановлення
  • Перезавантажте комп'ютер
  • Переконайтеся, що Node.js додано до змінної PATH

Завдання: Встановити Node.js на macOS

Варіант 1: Через офіційний інсталятор

  1. Відкрийте nodejs.org
  2. Завантажте версію LTS для macOS
  3. Відкрийте завантажений .pkg файл
  4. Слідуйте інструкціям інсталятора

Варіант 2: Через Homebrew (рекомендовано)

Terminal
# Якщо Homebrew не встановлений:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Встановлення Node.js:
brew install node

Перевірка встановлення:

Terminal
node -v
# Очікуваний результат: v20.x.x або v22.x.x

npm -v
# Очікуваний результат: 10.x.x

Завдання: Встановити Node.js на Linux/Ubuntu

Рекомендований спосіб — через NodeSource:

Terminal
# Оновіть пакети
sudo apt update

# Встановіть curl якщо немає
sudo apt install -y curl

# Додайте репозиторій NodeSource (Node.js 20.x)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Встановіть Node.js
sudo apt install -y nodejs

Альтернатива — через snap:

sudo snap install node --classic

Перевірка встановлення:

Terminal
node -v
# Очікуваний результат: v20.x.x або v22.x.x

npm -v
# Очікуваний результат: 10.x.x
Якщо є помилки з правами доступу (EACCES)

Не використовуйте sudo з npm. Натомість налаштуйте npm prefix:

mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
✅ Контрольна точка

Команди node -v та npm -v повертають версії без помилок.

1.2. Встановлення VS Code

Завдання: Встановити VS Code на Windows

  1. Перейдіть на code.visualstudio.com
  2. Натисніть "Download for Windows"
  3. Запустіть інсталятор VSCodeSetup-x64.exe
  4. Важливо! На кроці "Select Additional Tasks" відмітьте:
    • ✅ Add "Open with Code" action to Windows Explorer file context menu
    • ✅ Add "Open with Code" action to Windows Explorer directory context menu
    • ✅ Add to PATH
  5. Завершіть встановлення

Завдання: Встановити VS Code на macOS

  1. Перейдіть на code.visualstudio.com
  2. Завантажте версію для macOS (.zip)
  3. Розпакуйте архів
  4. Перетягніть Visual Studio Code.app в папку Applications
  5. Для команди code в терміналі:
    • Відкрийте VS Code
    • Натисніть Cmd + Shift + P
    • Введіть "Shell Command: Install 'code' command in PATH"
    • Виберіть цю опцію

Завдання: Встановити VS Code на Linux/Ubuntu

Через snap (найпростіше):

sudo snap install code --classic

Або через .deb пакет:

# Завантажте .deb з code.visualstudio.com
# Встановіть:
sudo dpkg -i code_*.deb
sudo apt install -f  # якщо є помилки залежностей

Рекомендовані розширення VS Code

Після встановлення VS Code, встановіть ці розширення (Extensions → Search):

  • ES7+ React/Redux/React-Native snippets — швидкі шаблони коду
  • Prettier - Code formatter — автоформатування коду
  • ESLint — перевірка помилок JavaScript
  • Tailwind CSS IntelliSense — автодоповнення Tailwind класів
  • Auto Rename Tag — автоматичне перейменування HTML тегів
✅ Контрольна точка

VS Code встановлений, розширення додані. Команда code . в терміналі відкриває VS Code.

1.3. Встановлення Git

Завдання: Встановити Git на Windows

  1. Перейдіть на git-scm.com/download/win
  2. Завантаження почнеться автоматично
  3. Запустіть інсталятор
  4. Під час встановлення:
    • Default editor: оберіть Visual Studio Code
    • Adjusting PATH: оберіть "Git from the command line and also from 3rd-party software"
    • Інші опції залиште за замовчуванням

Перевірка:

PowerShell (новий термінал!)
git --version
# Очікуваний результат: git version 2.x.x

Завдання: Встановити Git на macOS

Git зазвичай вже встановлений на macOS. Перевірте:

git --version

Якщо Git не встановлений, система запропонує встановити Command Line Tools.

Альтернатива через Homebrew:

brew install git

Завдання: Встановити Git на Linux/Ubuntu

sudo apt update
sudo apt install git

Перевірка:

git --version
# Очікуваний результат: git version 2.x.x

Налаштування Git (для всіх ОС)

Terminal / PowerShell
# Замініть на своє ім'я та email
git config --global user.name "Ваше Ім'я"
git config --global user.email "your.email@example.com"

# Перевірка налаштувань
git config --list
✅ Контрольна точка — Підготовка завершена

Всі три інструменти встановлені та працюють:

  • node -v → v20.x.x або новіше
  • npm -v → 10.x.x
  • git --version → git version 2.x.x
  • VS Code відкривається

2. Створення Next.js проекту

Завдання: Створити новий Next.js проект

Крок 1: Відкрийте термінал і перейдіть в папку для проектів

Terminal / PowerShell
# Windows (PowerShell)
cd C:\Users\ВашеІмя\Desktop
# або
cd ~\Desktop

# macOS / Linux
cd ~/Desktop
# або будь-яка інша папка для проектів

Крок 2: Створіть Next.js проект

Terminal / PowerShell
npx create-next-app@latest my-project
Примітка: Замініть my-project на назву вашого проекту (латиницею, без пробілів). Наприклад: library-app, cinema-system, hotel-booking.

Крок 3: Якщо система запитає про встановлення пакету — погодьтеся:

Запит на встановлення
Need to install the following packages:
  create-next-app@16.x.x
Ok to proceed? (y) › y

Крок 4: На питання про налаштування за замовчуванням оберіть "No":

Вибір режиму налаштування
? Would you like to use the recommended Next.js defaults? › No, customize settings

Крок 5: Відповідайте на питання:

Інтерактивне меню
✔ Would you like to use TypeScript? › No
✔ Which linter would you like to use? › ESLint
✔ Would you like to use Tailwind CSS? › Yes
✔ Would you like your code inside a `src/` directory? › No
✔ Would you like to use App Router? (recommended) › Yes
✔ Would you like to use React Compiler? › No
✔ Would you like to customize the import alias (@/* by default)? › No
Важливо: Обов'язково оберіть Yes для App Router та No для React Compiler!

Крок 4: Дочекайтеся завершення встановлення (1-5 хвилин)

Можливі помилки та їх вирішення
  • EACCES / Permission denied:
    Windows: запустіть PowerShell як Адміністратор
    Linux/macOS: НЕ використовуйте sudo, налаштуйте npm prefix
  • Network error / ETIMEDOUT:
    Перевірте інтернет-з'єднання або спробуйте пізніше
  • npm WARN deprecated:
    Це попередження, не помилка. Можна ігнорувати.

Завдання: Запустити dev сервер

Terminal / PowerShell
# Перейти в папку проекту
cd my-project

# Запустити dev сервер
npm run dev

Очікуваний результат:

   ▲ Next.js 15.x.x
   - Local:        http://localhost:3000

 ✓ Starting...
 ✓ Ready in 2.3s

Завдання: Відкрити проект в браузері

  1. Відкрийте браузер (Chrome, Firefox, Edge)
  2. Перейдіть за адресою: http://localhost:3000
  3. Ви побачите стартову сторінку Next.js

Завдання: Відкрити проект в VS Code

В новому терміналі (не закривайте попередній!):

# Переконайтеся, що ви в папці проекту
cd my-project

# Відкрийте VS Code
code .
✅ Контрольна точка
  • Dev сервер запущений і показує "Ready"
  • Сторінка localhost:3000 відкривається в браузері
  • Проект відкритий у VS Code

3. Створення Landing Page

Тепер ви створите landing page для вашого проекту. Нижче наведено приклад для кав'ярні — адаптуйте його під вашу тему!

Завдання: Замінити головну сторінку

  1. У VS Code відкрийте файл app/page.js
  2. Видаліть весь вміст файлу
  3. Вставте наступний код (та адаптуйте під свою тему):
app/page.js
// Головна сторінка вашого проекту
// Замініть назви та тексти на відповідні вашій темі!

export default function Home() {
  return (
    <div className="min-h-screen">
      {/* Hero секція */}
      <section className="bg-gradient-to-r from-blue-600 to-blue-800 text-white py-20">
        <div className="container mx-auto px-4 text-center">
          <h1 className="text-5xl font-bold mb-4">
            Назва вашого проекту
          </h1>
          <p className="text-xl mb-8 opacity-90">
            Короткий опис — що робить ваш застосунок (1-2 речення)
          </p>
          <button className="bg-white text-blue-600 px-8 py-3 rounded-lg font-semibold hover:bg-blue-100 transition">
            Розпочати
          </button>
        </div>
      </section>

      {/* Секція можливостей */}
      <section className="py-16 bg-gray-50">
        <div className="container mx-auto px-4">
          <h2 className="text-3xl font-bold text-center mb-12 text-gray-900">
            Можливості системи
          </h2>

          <div className="grid grid-cols-1 md:grid-cols-3 gap-8">
            {/* Картка 1 */}
            <div className="bg-white rounded-lg shadow-md p-6 hover:shadow-lg transition">
              <div className="text-4xl mb-4">📚</div>
              <h3 className="text-xl font-semibold mb-2 text-gray-900">Можливість 1</h3>
              <p className="text-gray-600">
                Опис першої можливості вашої системи.
              </p>
            </div>

            {/* Картка 2 */}
            <div className="bg-white rounded-lg shadow-md p-6 hover:shadow-lg transition">
              <div className="text-4xl mb-4">👥</div>
              <h3 className="text-xl font-semibold mb-2 text-gray-900">Можливість 2</h3>
              <p className="text-gray-600">
                Опис другої можливості вашої системи.
              </p>
            </div>

            {/* Картка 3 */}
            <div className="bg-white rounded-lg shadow-md p-6 hover:shadow-lg transition">
              <div className="text-4xl mb-4">🔒</div>
              <h3 className="text-xl font-semibold mb-2 text-gray-900">Можливість 3</h3>
              <p className="text-gray-600">
                Опис третьої можливості вашої системи.
              </p>
            </div>
          </div>
        </div>
      </section>

      {/* Футер */}
      <footer className="bg-gray-800 text-white py-8">
        <div className="container mx-auto px-4 text-center">
          <p className="text-gray-400">
            © 2024 Ваше Ім'я | Курс "Основи обробки та передачі інформації"
          </p>
        </div>
      </footer>
    </div>
  )
}
  1. Збережіть файл: Ctrl + S (Windows/Linux) або Cmd + S (macOS)
  2. Перевірте браузер — сторінка оновиться автоматично (Hot Reload)
Hot Reload: Одна з найзручніших функцій Next.js — при збереженні файлу зміни відразу відображаються в браузері без перезапуску сервера.

Завдання: Адаптувати під свою тему

Замініть у коді:

  • Назву проекту — відповідно до вашої теми (Бібліотека, Кінотеатр, Готель тощо)
  • Опис — що робить ваша система
  • Можливості — 3 головні функції вашого застосунку
  • Кольори — змініть blue на інший колір (red, green, purple, amber тощо)
  • Emoji — підберіть відповідні до вашої теми
  • Ваше ім'я — у футері

Завдання: Оновити metadata в layout.js

Відкрийте файл app/layout.js та знайдіть об'єкт metadata:

app/layout.js — знайдіть та змініть metadata
export const metadata = {
  title: "Назва вашого проекту",           // Відображається у вкладці браузера
  description: "Короткий опис проекту",    // Для SEO та соціальних мереж
};

Приклади для різних тем:

// Бібліотека
title: "Система управління бібліотекою"
description: "Зручний облік книг та читачів"

// Кінотеатр
title: "Кінотеатр «Зоряний»"
description: "Онлайн-бронювання квитків на найкращі фільми"

// Готель
title: "Готель «Затишок»"
description: "Система бронювання номерів"
Що таке metadata? Це інформація про сторінку, яка відображається:
  • У вкладці браузера (title)
  • При пошуку в Google (description)
  • При поширенні посилання в соцмережах

Приклади адаптації для різних тем

Приклади заголовків
// Бібліотека

4. Створення сторінки /about

Тепер створіть другу сторінку для вашого проекту — сторінку "Про нас" / "Про проект". В Next.js кожна папка всередині app/ з файлом page.js стає окремим маршрутом.

Завдання: Створити сторінку /about

  1. Створіть папку app/about/
  2. Всередині створіть файл page.js
  3. Додайте контент про ваш проект:
app/about/page.js
export default function AboutPage() {
  return (
    <div className="min-h-screen py-16">
      <div className="container mx-auto px-4">
        <h1 className="text-4xl font-bold mb-6 text-gray-900">
          Про проект
        </h1>
        <p className="text-lg text-gray-700 mb-4">
          Опишіть ваш проект — що це за система, для кого вона призначена,
          які проблеми вирішує.
        </p>
        <p className="text-lg text-gray-700">
          Додайте інформацію про технології, команду або будь-що
          релевантне до вашої теми.
        </p>
      </div>
    </div>
  )
}
Адаптація: Змініть заголовок та тексти відповідно до вашої теми. Наприклад, для бібліотеки — "Про бібліотеку", для кінотеатру — "Про кінотеатр".
✅ Контрольна точка
  • Сторінка /about відкривається в браузері
  • Має заголовок та абзац з реальним контентом
  • Контент відповідає вашій темі

5. Збереження та завантаження на GitHub

Завдання: Створити гілку та зробити коміт

Terminal (в папці проекту)
git checkout -b LR1
git add .
git commit -m "LR1: create Next.js project with landing page and about"

Якщо ви ще не створили репозиторій на GitHub:

  3. Після цього виконайте git push -u origin LR1

Підсумок

  • Node.js та npm встановлені і працюють
  • VS Code встановлений з рекомендованими розширеннями
  • Git встановлений та налаштований
  • Next.js проект створений та запускається
  • Landing page відображається коректно
  • Контент адаптований під обрану тему
  • Hot Reload працює (зміни застосовуються автоматично)
  • Сторінка /about створена з контентом відповідно до теми
