Мета заняття

На цьому практичному занятті ви:

  • Створите middleware для захисту маршрутів dashboard від неаутентифікованих користувачів
  • Захистите API endpoints (POST, PUT, DELETE) перевіркою авторизації
  • Створите хелпер-функцію authorize() для уніфікованої перевірки ролі
  • Реалізуєте сторінку управління користувачами (тільки для admin)
  • Додасте умовний рендеринг UI на основі ролі (badge, навігація, кнопки)
Передумова: Ви маєте мати працюючий Next.js проєкт з тижня 8 (NextAuth.js, реєстрація, логін, модель User з полем role). Ми продовжуємо працювати з тим самим проєктом.
Приклад викладача: Кав’ярня «Кофеїн». Ви маєте адаптувати код під свою тему з 15 варіантів. Наприклад, для «Бібліотеки» — замість напоїв будуть книги, замість /dashboard/drinks — /dashboard/books.

2. Створення Middleware

Middleware — це функція, яка виконується перед обробкою кожного запиту. Ми використаємо її для захисту маршрутів /dashboard — якщо користувач не увійшов у систему, він буде перенаправлений на сторінку логіну.

Завдання: Створити файл middleware.js

Створіть файл middleware.js в корені проєкту (на одному рівні з папкою app/, не всередині неї!):

middleware.js (корінь проєкту)
// Тиждень 9: Middleware для захисту маршрутів
// Перехоплює запити до /dashboard/* і перевіряє JWT-токен

import { NextResponse } from "next/server";
import { getToken } from "next-auth/jwt";

export async function middleware(request) {
  // Отримуємо JWT-токен із cookie
  const token = await getToken({
    req: request,
    secret: process.env.NEXTAUTH_SECRET,
  });

  // Якщо токена немає — redirect на сторінку логіну
  if (!token) {
    const loginUrl = new URL("/login", request.url);
    // Зберігаємо URL, куди хотів потрапити користувач
    loginUrl.searchParams.set("callbackUrl", request.url);
    return NextResponse.redirect(loginUrl);
  }

  // Токен є — пропускаємо запит далі
  return NextResponse.next();
}

// Конфігурація: які маршрути захищати
export const config = {
  matcher: ["/dashboard/:path*"],
};
Чому getToken, а не getServerSession? Middleware працює на Edge Runtime (не Node.js), де getServerSession не підтримується. Функція getToken з next-auth/jwt лише декодує JWT із cookie — це швидко і не потребує запиту до бази даних.
⚠️ Типова помилка: middleware не спрацьовує

Файл middleware.js обов’язково має бути в корені проєкту, поруч з папкою app/. Якщо розмістити його всередині app/ — Next.js його не знайде. Перевірте структуру:

Правильна структура
my-project/
  middleware.js    ← ТУТ (в корені!)
  app/
    page.js
    dashboard/
      page.js
  lib/
  components/

3. Хелпер авторизації

Перед тим як захищати API endpoints, створимо хелпер-функцію, яка перевірятиме сесію та роль користувача. Це дозволить уникнути дублювання коду в кожному API Route.

Завдання: Створити lib/authorize.js

lib/authorize.js
// Тиждень 9: Хелпер для перевірки авторизації в API Routes

import { getServerSession } from "next-auth";
import { authOptions } from "@/lib/auth";

/**
 * Перевіряє авторизацію користувача
 * @param {string|null} requiredRole — необхідна роль (null = будь-який аутентифікований)
 * @returns {{ session, error }} — сесія або Response з помилкою
 */
export async function authorize(requiredRole = null) {
  const session = await getServerSession(authOptions);

  // Не аутентифікований
  if (!session) {
    return {
      session: null,
      error: Response.json(
        { error: "Необхідно увійти в систему" },
        { status: 401 }
      ),
    };
  }

  // Перевірка ролі (якщо вказана)
  if (requiredRole && session.user.role !== requiredRole) {
    return {
      session: null,
      error: Response.json(
        { error: `Потрібна роль: ${requiredRole}` },
        { status: 403 }
      ),
    };
  }

  // Все ОК
  return { session, error: null };
}
Як використовувати authorize():
  • authorize() — будь-який аутентифікований користувач (user або admin)
  • authorize('admin') — тільки admin
  • Якщо error не null — повертаємо його як Response

4. Захист API endpoints

Зараз API endpoints для створення, редагування та видалення напоїв доступні всім без обмежень. Додамо перевірку авторизації: GET залишимо публічним, а POST/PUT/DELETE — тільки для admin.

Завдання: Захистити POST в app/api/drinks/route.js

Оновіть функцію POST — додайте перевірку авторизації на початку:

app/api/drinks/route.js (оновлений POST)
// Тиждень 9: додано захист API через authorize()

import dbConnect from "@/lib/db";
import Drink from "@/lib/models/Drink";
import { authorize } from "@/lib/authorize";

// GET — публічний (без змін)
export async function GET(request) {
  await dbConnect();

  const { searchParams } = new URL(request.url);
  const category = searchParams.get("category");
  const search = searchParams.get("search");

  const filter = {};

  if (category && category !== "Всі") {
    filter.category = category;
  }

  if (search) {
    filter.$or = [
      { name: { $regex: search, $options: "i" } },
      { description: { $regex: search, $options: "i" } },
    ];
  }

  const drinks = await Drink.find(filter).sort({ createdAt: -1 });
  return Response.json(drinks);
}

// POST — тільки admin
export async function POST(request) {
  // Перевірка авторизації
  const { session, error } = await authorize("admin");
  if (error) return error;

  await dbConnect();

  try {
    const data = await request.json();
    const drink = await Drink.create(data);

    return Response.json(drink, { status: 201 });
  } catch (error) {
    if (error.name === "ValidationError") {
      const messages = Object.values(error.errors).map((err) => err.message);
      return Response.json({ errors: messages }, { status: 400 });
    }

    return Response.json({ error: "Помилка сервера" }, { status: 500 });
  }
}

Завдання: Захистити PUT та DELETE в app/api/drinks/[id]/route.js

Додайте перевірку авторизації до PUT та DELETE. GET залишіть без змін:

app/api/drinks/[id]/route.js (оновлений)
// Тиждень 9: додано захист PUT та DELETE через authorize()

import dbConnect from "@/lib/db";
import Drink from "@/lib/models/Drink";
import { authorize } from "@/lib/authorize";

// GET — публічний (без змін)
export async function GET(request, { params }) {
  await dbConnect();
  const { id } = await params;

  try {
    const drink = await Drink.findById(id);

    if (!drink) {
      return Response.json({ error: "Напій не знайдено" }, { status: 404 });
    }

    return Response.json(drink);
  } catch (error) {
    return Response.json({ error: "Невалідний ID" }, { status: 400 });
  }
}

// PUT — тільки admin
export async function PUT(request, { params }) {
  const { session, error } = await authorize("admin");
  if (error) return error;

  await dbConnect();
  const { id } = await params;

  try {
    const data = await request.json();
    const drink = await Drink.findByIdAndUpdate(id, data, {
      new: true,
      runValidators: true,
    });

    if (!drink) {
      return Response.json({ error: "Напій не знайдено" }, { status: 404 });
    }

    return Response.json(drink);
  } catch (error) {
    if (error.name === "ValidationError") {
      const messages = Object.values(error.errors).map((err) => err.message);
      return Response.json({ errors: messages }, { status: 400 });
    }

    return Response.json({ error: "Помилка сервера" }, { status: 500 });
  }
}

// DELETE — тільки admin
export async function DELETE(request, { params }) {
  const { session, error } = await authorize("admin");
  if (error) return error;

  await dbConnect();
  const { id } = await params;

  try {
    const drink = await Drink.findByIdAndDelete(id);

    if (!drink) {
      return Response.json({ error: "Напій не знайдено" }, { status: 404 });
    }

    return Response.json({ message: `Напій "${drink.name}" видалено` });
  } catch (error) {
    return Response.json({ error: "Невалідний ID" }, { status: 400 });
  }
}
✅ Контрольна точка

Перевірте через Thunder Client або Postman:

  • GET /api/drinks — працює без логіну (200)
  • POST /api/drinks без cookie — повертає 401
  • DELETE /api/drinks/будь-який-id без cookie — повертає 401
  • Увійдіть як user — POST повертає 403 («Потрібна роль: admin»)

5. Сторінка управління користувачами

Створимо сторінку, де адміністратор може переглядати список всіх зареєстрованих користувачів. Ця сторінка буде доступна тільки для ролі admin.

Завдання: Створити API для списку користувачів

Створіть файл app/api/users/route.js:

app/api/users/route.js
// Тиждень 9: API для списку користувачів (тільки admin)

import dbConnect from "@/lib/db";
import User from "@/lib/models/User";
import { authorize } from "@/lib/authorize";

// GET /api/users — список користувачів (тільки admin)
export async function GET() {
  const { session, error } = await authorize("admin");
  if (error) return error;

  await dbConnect();

  // Отримуємо всіх користувачів без паролів
  const users = await User.find()
    .select("-password")
    .sort({ createdAt: -1 });

  return Response.json(users);
}

Чому перевірка і в middleware, і на сторінці? Middleware захищає від неаутентифікованих (немає токена → redirect на /login). Сторінка додатково перевіряє роль (user → redirect на /dashboard). Це два незалежних рівні захисту (defence in depth).

6. Зміна ролі користувача

Адміністратор повинен мати можливість змінювати роль інших користувачів прямо зі сторінки управління. Для цього створимо API endpoint та клієнтський компонент з кнопкою зміни ролі.

Завдання: Створити API для зміни ролі

Створіть файл app/api/users/[id]/route.js:

app/api/users/[id]/route.js
// Тиждень 9: API для зміни ролі користувача (тільки admin)

import dbConnect from "@/lib/db";
import User from "@/lib/models/User";
import { authorize } from "@/lib/authorize";

// PUT /api/users/[id] — зміна ролі
export async function PUT(request, { params }) {
  const { session, error } = await authorize("admin");
  if (error) return error;

  await dbConnect();
  const { id } = await params;

  try {
    const { role } = await request.json();

    // Валідація ролі
    if (!["user", "admin"].includes(role)) {
      return Response.json(
        { error: "Роль має бути 'user' або 'admin'" },
        { status: 400 }
      );
    }

    // Не дозволяємо змінити роль самому собі
    if (id === session.user.id) {
      return Response.json(
        { error: "Не можна змінити власну роль" },
        { status: 400 }
      );
    }

    const user = await User.findByIdAndUpdate(
      id,
      { role },
      { new: true }
    ).select("-password");

    if (!user) {
      return Response.json(
        { error: "Користувача не знайдено" },
        { status: 404 }
      );
    }

    return Response.json(user);
  } catch (error) {
    return Response.json({ error: "Помилка сервера" }, { status: 500 });
  }
}
Захист від самозміни: Адмін не може змінити власну роль — це захист від випадкового «зниження» себе до user, після чого ніхто не зможе керувати системою.

Завдання: Створити компонент кнопки зміни ролі

Створіть файл components/RoleToggle.jsx:

components/RoleToggle.jsx
// Тиждень 9: Кнопка зміни ролі користувача (Client Component)

"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";

export default function RoleToggle({ userId, currentRole, currentUserId }) {
  const [loading, setLoading] = useState(false);
  const router = useRouter();

  // Не показуємо кнопку для поточного користувача
  if (userId === currentUserId) {
    return <span className="text-xs text-gray-400">(ви)</span>;
  }

  const newRole = currentRole === "admin" ? "user" : "admin";

  const handleToggle = async () => {
    setLoading(true);

    try {
      const response = await fetch(`/api/users/${userId}`, {
        method: "PUT",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ role: newRole }),
      });

      if (!response.ok) {
        const data = await response.json();
        alert(data.error || "Помилка зміни ролі");
        return;
      }

      // Оновлюємо сторінку для відображення нової ролі
      router.refresh();
    } catch (error) {
      alert("Помилка з'єднання");
    } finally {
      setLoading(false);
    }
  };

  return (
    <button
      onClick={handleToggle}
      disabled={loading}
      className="text-xs px-2 py-1 rounded bg-gray-100 hover:bg-gray-200
        text-gray-600 transition disabled:opacity-50 cursor-pointer"
    >
      {loading ? "..." : `→ ${newRole}`}
    </button>
  );
}

                    userId={user._id.toString()}

            ))}
          </tbody>

      </div>

      </p>
    </div>
  );
}

'use client'
import Link from "next/link";

import { useSession } from "next-auth/react";

export default function DashboardNav() {

    { href: "/dashboard/drinks", label: "Напої" },
    // Пункт "Користувачі" тільки для admin

  ];

  return (
    <nav>
      <ul className="space-y-2">
        {links.map((link) => {
          const isActive =
            link.href === "/dashboard"
              ? pathname === "/dashboard"
              : pathname.startsWith(link.href);

          return (
            <li key={link.href}>
              <Link
                href={link.href}
                className={`block px-4 py-2 rounded transition-colors ${
                  isActive
                    ? "bg-amber-700 text-white"
                    : "text-gray-300 hover:bg-gray-700"
                }`}
              >
                {link.label}
              </Link>
            </li>
          );
        })}
      </ul>
    </nav>

}
