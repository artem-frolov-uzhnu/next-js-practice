1. Fetching даних з API

В Next.js Server Components можна використовувати async/await для fetching даних безпосередньо в компоненті — без useState, useEffect чи бібліотек. Це простіше, швидше та безпечніше.

Завдання: Створити сторінку з постами від JSONPlaceholder

Створіть файл app/demo/posts/page.js — сторінка, що отримує дані з публічного API:

app/demo/posts/page.js
export default async function PostsPage() {
  const response = await fetch('https://jsonplaceholder.typicode.com/posts')
  const posts = await response.json()

  return (
    <div className="container mx-auto px-4 py-8">
      <h1 className="text-4xl font-bold mb-8">Пости з JSONPlaceholder</h1>

      <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
        {posts.slice(0, 9).map(post => (
          <article
            key={post.id}
            className="bg-white rounded-lg shadow p-6 hover:shadow-lg transition-shadow"
          >
            <h2 className="text-xl font-bold mb-2 line-clamp-2">
              {post.title}
            </h2>
            <p className="text-gray-600 line-clamp-3">
              {post.body}
            </p>
            <div className="mt-4 text-sm text-gray-500">
              Post #{post.id} by User {post.userId}
            </div>
          </article>
        ))}
      </div>
    </div>
  )
}
Зверніть увагу: Компонент async — це Server Component. Дані завантажуються на сервері, користувач отримує готовий HTML. Жодного useState, useEffect, loading state — все простіше!
✅ Контрольна точка
  • Відкрийте /demo/posts — має відображатись 9 постів у сітці
  • Дані завантажені на сервері (перевірте View Source — контент є в HTML)

2. Loading States та Skeleton UI

Next.js автоматично показує loading.js при навігації на сторінку, поки page.js завантажує дані. Це працює без жодного клієнтського коду — просто створіть файл loading.js поруч з page.js.

Завдання: Додати loading для сторінки постів

Створіть app/demo/posts/loading.js — skeleton, що повторює структуру сітки постів:

app/demo/posts/loading.js
export default function Loading() {
  return (
    <div className="container mx-auto px-4 py-8">
      <div className="h-10 bg-gray-200 rounded w-64 mb-8 animate-pulse"></div>

      <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
        {[...Array(9)].map((_, i) => (
          <div key={i} className="bg-white rounded-lg shadow p-6">
            <div className="h-6 bg-gray-200 rounded w-3/4 mb-2 animate-pulse"></div>
            <div className="h-4 bg-gray-200 rounded w-full mb-1 animate-pulse"></div>
            <div className="h-4 bg-gray-200 rounded w-5/6 mb-1 animate-pulse"></div>
            <div className="h-4 bg-gray-200 rounded w-4/5 animate-pulse"></div>
            <div className="h-3 bg-gray-200 rounded w-32 mt-4 animate-pulse"></div>
          </div>
        ))}
      </div>
    </div>
  )
}
Принцип: Skeleton має повторювати структуру реального контенту. 9 карток у сітці 3 колонки — як і на реальній сторінці. Клас animate-pulse додає плавну анімацію мерехтіння.

'use client'

import { useEffect } from 'react'

export default function Error({ error, reset }) {
  useEffect(() => {
    console.error('Error caught:', error)
  }, [error])

  return (
    <div className="min-h-screen flex items-center justify-center bg-red-50">
      <div className="max-w-md w-full bg-white rounded-lg shadow-lg p-8">
        <div className="text-center">

            Виникла помилка
          </h2>
          <p className="text-gray-600 mb-6">
            {error.message}
          </p>
          <button
            onClick={reset}
            className="bg-blue-600 text-white px-6 py-3 rounded hover:bg-blue-700 font-bold"
          >
            Спробувати знову
          </button>
        </div>
      </div>
    </div>
  )
}
⚠️ Важливо: error.js ОБОВ'ЯЗКОВО має 'use client'

Файл error.js завжди має бути Client Component з директивою 'use client' на початку файлу. Це вимога Next.js, тому що error boundary використовує React hooks (useEffect) та інтерактивні елементи (кнопка reset). Без цієї директиви ви отримаєте помилку.

const users = {
  '1': { id: 1, name: 'John Doe' },

      <Suspense fallback={
        <div className="grid grid-cols-3 gap-4 mb-8">
          <StatSkeleton />
          <StatSkeleton />
          <StatSkeleton />
        </div>
      }>
        <FastStats />
      </Suspense>

      {/* Повільний контент */}
      <Suspense fallback={
        <div className="space-y-4">
          <div className="h-8 bg-gray-200 rounded w-48 animate-pulse mb-4"></div>
          <PostSkeleton />
          <PostSkeleton />
          <PostSkeleton />
        </div>
      }>
        <SlowPosts />
      </Suspense>
    </div>
  )
}
Як працює Streaming: Next.js відправляє HTML частинами. Спочатку заголовок та skeleton — через 0.5 сек FastStats замінює свій skeleton — через 3 сек SlowPosts замінює свій. Кожна секція Suspense незалежна!
