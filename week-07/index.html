1. Реєстрація та налаштування MongoDB Atlas

MongoDB Atlas — це хмарний сервіс, який дозволяє створити базу даних MongoDB без встановлення на локальний комп’ютер. Безплатний рівень (M0 Free) надає 512 МБ — більш ніж достатньо для нашого навчального проекту.

Завдання: Створити акаунт та кластер на MongoDB Atlas

  1. Перейдіть на mongodb.com/cloud/atlas/register
  2. Зареєструйтесь (можна через Google акаунт)
  3. Створіть організацію та проєкт (назва — будь-яка, наприклад nextjs-course)
  4. Натисніть Build a Database (або Create a Deployment)
  5. Оберіть M0 FREE
  6. Оберіть провайдера (AWS) та найближчий регіон (наприклад, eu-central-1 Frankfurt)
  7. Натисніть Create Deployment

Завдання: Налаштувати доступ до бази

Після створення кластеру Atlas запропонує налаштувати безпеку:

  1. Database User — створіть користувача:
    • Username: наприклад, student
    • Password: згенеруйте або придумайте надійний пароль
    • Збережіть логін та пароль — вони знадобляться для connection string
  2. Network Access — додайте IP-адресу:
    • Натисніть Add My Current IP Address для додавання вашої IP
    • Або додайте 0.0.0.0/0 (доступ звідусіль — зручно для навчання)
⚠️ Безпека

0.0.0.0/0 дозволяє підключення з будь-якої IP-адреси. Це прийнятно для навчального проекту, але неприпустимо для продакшену. В реальних проектах завжди обмежуйте доступ конкретними IP-адресами.

Завдання: Отримати Connection String

  1. У розділі Database натисніть Connect поруч із вашим кластером
  2. Оберіть Drivers
  3. Скопіюйте connection string — він виглядає приблизно так:
Connection String (приклад)
mongodb+srv://student:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority

Замініть YOUR_PASSWORD на ваш реальний пароль і додайте назву бази даних перед ?:

Connection String з назвою бази
mongodb+srv://student:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/coffein?retryWrites=true&w=majority

Назва бази (тут coffein) — MongoDB створить її автоматично при першому записі.

✅ Контрольна точка
  • Ви маєте акаунт на MongoDB Atlas
  • Кластер M0 FREE створено
  • Database User створено (збережіть логін/пароль)
  • Network Access налаштовано
  • Connection string скопійовано

2. Підключення MongoDB до Next.js

Завдання: Встановити Mongoose

Terminal (в папці проекту)
npm install mongoose

Завдання: Створити файл змінних середовища

Створіть файл .env.local в корені проекту (поруч з package.json):

.env.local
MONGODB_URI=mongodb+srv://student:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/coffein?retryWrites=true&w=majority

Замініть значення на ваш реальний connection string.

Важливо! Переконайтесь, що .env.local є в .gitignore. Next.js додає його автоматично при створенні проекту через create-next-app. Перевірте: відкрийте .gitignore і знайдіть рядок .env*.local.

Завдання: Створити утиліту підключення до БД

Створіть файл lib/db.js:

lib/db.js
import mongoose from 'mongoose'

const MONGODB_URI = process.env.MONGODB_URI

if (!MONGODB_URI) {
  throw new Error(
    'Будь ласка, додайте MONGODB_URI до файлу .env.local'
  )
}

let cached = global.mongoose

if (!cached) {
  cached = global.mongoose = { conn: null, promise: null }
}

async function dbConnect() {
  if (cached.conn) {
    return cached.conn
  }

  if (!cached.promise) {
    const opts = {
      bufferCommands: false,
    }

    cached.promise = mongoose.connect(MONGODB_URI, opts)
      .then((mongoose) => mongoose)
  }

  try {
    cached.conn = await cached.promise
  } catch (e) {
    cached.promise = null
    throw e
  }

  return cached.conn
}

export default dbConnect
Навіщо global.mongoose? В development-режимі Next.js перезавантажує модулі при кожній зміні коду (Hot Reload). Без кешування кожен запит створюватиме нове з’єднання з базою, що швидко вичерпає ліміт. Об’єкт global зберігається між перезавантаженнями.

Завдання: Перевірити підключення

Створіть тимчасовий тестовий Route Handler для перевірки з’єднання:

app/api/test/route.js (оновіть існуючий)
import dbConnect from '@/lib/db'

export async function GET() {
  try {
    await dbConnect()

    return Response.json({
      message: 'MongoDB підключено!',
      status: 'ok',
      timestamp: new Date().toISOString()
    })
  } catch (error) {
    return Response.json({
      message: 'Помилка підключення до MongoDB',
      error: error.message,
    }, { status: 500 })
  }
}
✅ Контрольна точка
  • Запустіть сервер: npm run dev
  • Відкрийте http://localhost:3000/api/test
  • Ви маєте побачити JSON з "message": "MongoDB підключено!"
  • Якщо бачите помилку — перевірте connection string в .env.local (перезапустіть сервер після зміни!)
⚠️ Типова помилка: «MONGODB_URI is not defined»

Якщо ви бачите помилку про відсутність MONGODB_URI, переконайтесь що: (1) файл називається саме .env.local, (2) він у корені проекту, (3) ви перезапустили dev-сервер (Ctrl+C, потім npm run dev). Next.js читає .env.local тільки при запуску.

3. Створення Mongoose моделі

Mongoose модель визначає структуру документа в MongoDB: які поля він має, якого типу, які обов’язкові, та які правила валідації.

Завдання: Створити модель Drink

Створіть файл lib/models/Drink.js:

lib/models/Drink.js
import mongoose from 'mongoose'

const drinkSchema = new mongoose.Schema({
  name: {
    type: String,
    required: [true, 'Назва напою обов\'язкова'],
    trim: true,
    maxlength: [100, 'Назва не може бути довшою за 100 символів'],
  },
  description: {
    type: String,
    default: '',
    trim: true,
  },
  price: {
    type: Number,
    required: [true, 'Ціна обов\'язкова'],
    min: [0, 'Ціна не може бути від\'ємною'],
  },
  category: {
    type: String,
    enum: {
      values: ['hot', 'cold'],
      message: 'Категорія має бути: hot або cold',
    },
    default: 'hot',
  },
  available: {
    type: Boolean,
    default: true,
  },
}, {
  timestamps: true,
})

export default mongoose.models.Drink || mongoose.model('Drink', drinkSchema)
Зверніть увагу:
  • required: [true, 'повідомлення'] — поле обов’язкове з кастомним повідомленням
  • trim: true — автоматично видаляє пробіли на початку та в кінці
  • enum — обмежує допустимі значення
  • timestamps: true — автоматично додає createdAt і updatedAt
  • mongoose.models.Drink || ... — захист від помилки при Hot Reload
Для свого варіанту: Адаптуйте поля моделі під вашу тему. Наприклад, для «Онлайн-кінотеатру» це може бути Movie з полями title, genre, year, rating. Дивіться специфікацію свого варіанту в розділі «Варіанти проектів».
✅ Контрольна точка
  • Файл lib/models/Drink.js створено
  • Схема має щонайменше 3-4 поля з валідацією
  • Використовується timestamps: true
  • Є захист від Hot Reload (mongoose.models.Drink || ...)

5. Переведення API Routes на MongoDB

Тепер замінимо in-memory сховище (lib/drinks.js) на справжні операції з MongoDB через Mongoose. Структура ендпоінтів залишається тією ж — змінюється лише спосіб роботи з даними.

Завдання: Оновити GET та POST для списку

Оновіть файл app/api/drinks/route.js:

app/api/drinks/route.js
import dbConnect from '@/lib/db'
import Drink from '@/lib/models/Drink'

// GET /api/drinks
// GET /api/drinks?category=hot
// GET /api/drinks?search=лате
export async function GET(request) {
  await dbConnect()

  const { searchParams } = new URL(request.url)
  const category = searchParams.get('category')
  const search = searchParams.get('search')

  // Будуємо фільтр
  const filter = {}
  if (category) {
    filter.category = category
  }
  if (search) {
    filter.name = { $regex: search, $options: 'i' }
  }

  const drinks = await Drink.find(filter).sort({ createdAt: -1 })

  return Response.json({
    count: drinks.length,
    drinks,
  })
}

// POST /api/drinks
export async function POST(request) {
  await dbConnect()

  try {
    const body = await request.json()
    const drink = await Drink.create(body)

    return Response.json(drink, { status: 201 })
  } catch (error) {
    if (error.name === 'ValidationError') {
      const messages = Object.values(error.errors)
        .map(err => err.message)
      return Response.json({ errors: messages }, { status: 400 })
    }

    return Response.json(
      { error: 'Помилка сервера' },
      { status: 500 }
    )
  }
}
Що змінилось порівняно з тижнем 6:
  • Додано await dbConnect() на початку кожного обробника
  • Замість getAllDrinks() використовуємо Drink.find(filter)
  • Пошук через $regex з опцією i (регістронезалежний)
  • Замість createDrink() використовуємо Drink.create(body)
  • Валідація відбувається автоматично в Mongoose — обробляємо ValidationError

Завдання: Оновити GET, PUT, DELETE для одного напою

Оновіть файл app/api/drinks/[id]/route.js:

app/api/drinks/[id]/route.js
import dbConnect from '@/lib/db'
import Drink from '@/lib/models/Drink'

// GET /api/drinks/:id
export async function GET(request, { params }) {
  await dbConnect()
  const { id } = await params

  try {
    const drink = await Drink.findById(id)

    if (!drink) {
      return Response.json(
        { error: 'Напій не знайдено' },
        { status: 404 }
      )
    }

    return Response.json(drink)
  } catch (error) {
    // Невалідний формат ObjectId
    return Response.json(
      { error: 'Невалідний ID' },
      { status: 400 }
    )
  }
}

// PUT /api/drinks/:id
export async function PUT(request, { params }) {
  await dbConnect()
  const { id } = await params

  try {
    const body = await request.json()
    const drink = await Drink.findByIdAndUpdate(id, body, {
      new: true,
      runValidators: true,
    })

    if (!drink) {
      return Response.json(
        { error: 'Напій не знайдено' },
        { status: 404 }
      )
    }

    return Response.json(drink)
  } catch (error) {
    if (error.name === 'ValidationError') {
      const messages = Object.values(error.errors)
        .map(err => err.message)
      return Response.json({ errors: messages }, { status: 400 })
    }

    return Response.json(
      { error: 'Помилка сервера' },
      { status: 500 }
    )
  }
}

// DELETE /api/drinks/:id
export async function DELETE(request, { params }) {
  await dbConnect()
  const { id } = await params

  try {
    const drink = await Drink.findByIdAndDelete(id)

    if (!drink) {
      return Response.json(
        { error: 'Напій не знайдено' },
        { status: 404 }
      )
    }

    return Response.json({
      message: `Напій "${drink.name}" видалено`
    })
  } catch (error) {
    return Response.json(
      { error: 'Невалідний ID' },
      { status: 400 }
    )
  }
}
Важливо: Зверніть увагу на { new: true, runValidators: true } в findByIdAndUpdate():
  • new: true — повертає оновлений документ (за замовчуванням повертається старий)
  • runValidators: true — перевіряє валідацію при оновленні (за замовчуванням вимкнено!)

✓ Завдання: Оновити lib/helpers.js для роботи з MongoDB

Файл lib/helpers.js містить функцію getDrinkStats(), яку використовує сторінка Dashboard. Зараз вона імпортує дані з lib/drinks.js (in-memory масив). Потрібно переписати її для роботи з MongoDB:

lib/helpers.js
// Допоміжні функції для обчислення статистики
// Тиждень 7: використовує Mongoose замість in-memory масиву

import dbConnect from "./db";
import Drink from "./models/Drink";

export async function getDrinkStats() {
  await dbConnect();

  const drinks = await Drink.find();
  const total = drinks.length;
  const available = drinks.filter((d) => d.available).length;
  const unavailable = total - available;
  const categories = [...new Set(drinks.map((d) => d.category))];
  const avgPrice =
    total > 0
      ? Math.round(drinks.reduce((sum, d) => sum + d.price, 0) / total)
      : 0;

  return {
    total,
    available,
    unavailable,
    categoriesCount: categories.length,
    avgPrice,
  };
}

Зверніть увагу: функція тепер async, бо робить запит до бази даних. Якщо ваш app/dashboard/page.js викликає getDrinkStats(), переконайтесь, що виклик використовує await:

app/dashboard/page.js (фрагмент)
import { getDrinkStats } from "@/lib/helpers";

export default async function DashboardPage() {
  const stats = await getDrinkStats();  // ← await обов'язковий!
  // ...
}

6. Перевірка даних у MongoDB Compass

MongoDB Compass — безплатний GUI-клієнт для візуальної роботи з MongoDB. Він дозволяє переглядати, редагувати та аналізувати дані без написання коду.

Завдання: Встановити та підключити MongoDB Compass

  1. Завантажте MongoDB Compass з mongodb.com/try/download/compass
  2. Встановіть та відкрийте програму
  3. У полі «New Connection» вставте ваш connection string з .env.local
  4. Натисніть Connect
  5. Зліва ви побачите вашу базу даних (coffein) та колекцію (drinks)
  6. Натисніть на колекцію — побачите всі документи з їх полями
✅ Контрольна точка
  • MongoDB Compass підключено до вашого кластеру
  • Ви бачите базу даних та колекцію
  • Документи відображаються з усіма полями, включаючи _id, createdAt, updatedAt
  • Спробуйте створити/видалити напій через API та оновити вигляд у Compass
Альтернатива: Якщо не хочете встановлювати Compass, можна переглядати дані безпосередньо в Atlas: Database → Browse Collections.

7. Сторінка редагування напою

Наразі ми маємо сторінки для перегляду списку, деталей, створення та видалення напоїв. Залишилося додати можливість редагування — щоб кнопка «Редагувати» справді працювала. Для цього ми виділимо форму в окремий reusable компонент і використаємо його як для створення, так і для редагування.

Завдання: Створити reusable компонент DrinkForm

Створіть файл components/DrinkForm.jsx:

components/DrinkForm.jsx
'use client'

import { useState } from 'react'
import Link from 'next/link'

const CATEGORIES = ['Кава', 'Чай', 'Їжа', 'Інше']

export default function DrinkForm({
  initialData, onSubmit, submitLabel = 'Зберегти',
  isSubmitting, error
}) {
  const [formData, setFormData] = useState({
    name: initialData?.name || '',
    category: initialData?.category || '',
    price: initialData?.price || '',
    description: initialData?.description || '',
  })

  const handleChange = (e) => {
    setFormData({ ...formData, [e.target.name]: e.target.value })
  }

  const handleSubmit = (e) => {
    e.preventDefault()
    onSubmit({ ...formData, price: Number(formData.price) })
  }

  return (
    <>
      {error && (
        <div className="bg-red-50 border border-red-200
          text-red-700 px-4 py-3 rounded mb-6">
          {error}
        </div>
      )}

      <form onSubmit={handleSubmit} className="space-y-6">
        <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
          <div>
            <label className="block text-gray-700 font-bold mb-2">
              Назва *
            </label>
            <input type="text" name="name"
              value={formData.name} onChange={handleChange}
              required className="w-full px-4 py-2 border rounded
              focus:outline-none focus:border-amber-500" />
          </div>
          <div>
            <label className="block text-gray-700 font-bold mb-2">
              Категорія *
            </label>
            <select name="category"
              value={formData.category} onChange={handleChange}
              required className="w-full px-4 py-2 border rounded
              focus:outline-none focus:border-amber-500">
              <option value="">Оберіть категорію</option>
              {CATEGORIES.map((cat) => (
                <option key={cat} value={cat}>{cat}</option>
              ))}
            </select>
          </div>
          <div>
            <label className="block text-gray-700 font-bold mb-2">
              Ціна (грн) *
            </label>
            <input type="number" name="price"
              value={formData.price} onChange={handleChange}
              required className="w-full px-4 py-2 border rounded
              focus:outline-none focus:border-amber-500" />
          </div>
        </div>

        <div>
          <label className="block text-gray-700 font-bold mb-2">
            Опис
          </label>
          <textarea name="description"
            value={formData.description} onChange={handleChange}
            rows="4" className="w-full px-4 py-2 border rounded
            focus:outline-none focus:border-amber-500" />
        </div>

        <div className="flex gap-4">
          <button type="submit" disabled={isSubmitting}
            className="bg-amber-700 text-white px-6 py-3 rounded
            hover:bg-amber-800 font-bold disabled:opacity-50">
            {isSubmitting ? 'Збереження...' : submitLabel}
          </button>
          <Link href="/dashboard/drinks"
            className="bg-gray-300 text-gray-700 px-6 py-3
            rounded hover:bg-gray-400 font-bold inline-block">
            Скасувати
          </Link>
        </div>
      </form>
    </>
  )
}
Що таке reusable компонент? Замість копіювання однакової форми на двох сторінках (створення і редагування), ми виносимо її в окремий компонент DrinkForm. Він приймає initialData для редагування (або нічого для створення) і onSubmit для обробки відправки. Зверніть увагу: категорія тепер — <select> замість текстового поля, щоб відповідати enum у Mongoose-схемі.

app/dashboard/drinks/new/page.js
'use client'

import { useState } from 'react'
import { useRouter } from 'next/navigation'
import Link from 'next/link'
import DrinkForm from '@/components/DrinkForm'

export default function NewDrinkPage() {
  const router = useRouter()
  const [error, setError] = useState(null)
  const [isSubmitting, setIsSubmitting] = useState(false)

  const handleSubmit = async (formData) => {
    setError(null)
    setIsSubmitting(true)

    try {
      const response = await fetch('/api/drinks', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(formData),
      })

      if (!response.ok) {
        const data = await response.json()
        throw new Error(
          data.errors?.join(', ') || data.error || 'Помилка створення'
        )
      }

      router.push('/dashboard/drinks')
    } catch (err) {
      setError(err.message)
      setIsSubmitting(false)
    }
  }

  return (
    <div>
      <Link href="/dashboard/drinks"
        className="text-amber-700 hover:underline mb-4 inline-block">
        &larr; Назад до списку
      </Link>

      <div className="bg-white rounded-lg shadow p-8">
        <h1 className="text-3xl font-bold mb-6 text-gray-900">
          Додати новий напій
        </h1>
        <DrinkForm
          onSubmit={handleSubmit}
          submitLabel="Створити"
          isSubmitting={isSubmitting}
          error={error}
        />
      </div>
    </div>
  )
}

app/dashboard/drinks/[id]/edit/page.js
'use client'

import { useState, useEffect } from 'react'
import { useParams, useRouter } from 'next/navigation'
import Link from 'next/link'
import DrinkForm from '@/components/DrinkForm'

export default function EditDrinkPage() {
  const { id } = useParams()
  const router = useRouter()
  const [drink, setDrink] = useState(null)
  const [loading, setLoading] = useState(true)
  const [loadError, setLoadError] = useState(null)
  const [submitError, setSubmitError] = useState(null)
  const [isSubmitting, setIsSubmitting] = useState(false)

  useEffect(() => {
    fetch(`/api/drinks/${id}`)
      .then((res) => {
        if (!res.ok) throw new Error('Напій не знайдено')
        return res.json()
      })
      .then((data) => {
        setDrink(data)
        setLoading(false)
      })
      .catch((err) => {
        setLoadError(err.message)
        setLoading(false)
      })
  }, [id])

  const handleSubmit = async (formData) => {
    setSubmitError(null)
    setIsSubmitting(true)

    try {
      const response = await fetch(`/api/drinks/${id}`, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(formData),
      })

      if (!response.ok) {
        const data = await response.json()
        throw new Error(
          data.errors?.join(', ') || data.error || 'Помилка оновлення'
        )
      }

      router.push(`/dashboard/drinks/${id}`)
    } catch (err) {
      setSubmitError(err.message)
      setIsSubmitting(false)
    }
  }

  if (loading) {
    return (
      <div>
        <div className="h-4 bg-gray-200 rounded w-32
          mb-4 animate-pulse"></div>
        <div className="bg-white rounded-lg shadow p-8">
          <div className="h-8 bg-gray-200 rounded w-48
            animate-pulse"></div>
        </div>
      </div>
    )
  }

  if (loadError) {
    return (
      <div>
        <Link href="/dashboard/drinks"
          className="text-amber-700 hover:underline mb-4
          inline-block">
          &larr; Назад до списку
        </Link>
        <div className="bg-red-50 border border-red-200
          rounded-lg p-6 text-center">
          <h2 className="text-2xl font-bold text-red-600 mb-2">
            Помилка
          </h2>
          <p className="text-gray-600">{loadError}</p>
        </div>
      </div>
    )
  }

  return (
    <div>
      <Link href={`/dashboard/drinks/${id}`}
        className="text-amber-700 hover:underline mb-4
        inline-block">
        &larr; Назад до напою
      </Link>

      <div className="bg-white rounded-lg shadow p-8">
        <h1 className="text-3xl font-bold mb-6 text-gray-900">
          Редагувати: {drink.name}
        </h1>
        <DrinkForm
          initialData={drink}
          onSubmit={handleSubmit}
          submitLabel="Зберегти зміни"
          isSubmitting={isSubmitting}
          error={submitError}
        />
      </div>
    </div>
  )
}

Підсумок

Критерії успішного виконання

  • MongoDB Atlas: кластер створено, Database User та Network Access налаштовано
  • Mongoose встановлено (npm install mongoose)
  • .env.local створено з MONGODB_URI (не в Git!)
  • lib/db.js: утиліта підключення з кешуванням
  • lib/models/Drink.js: модель зі схемою та валідацією
  • GET /api/seed наповнює базу початковими даними
  • GET /api/drinks повертає дані з MongoDB
  • GET /api/drinks?category= фільтрація працює
  • GET /api/drinks/[id] повертає один документ
  • POST /api/drinks створює з Mongoose валідацією
  • PUT /api/drinks/[id] оновлює з runValidators: true
  • DELETE /api/drinks/[id] видаляє документ
  • Дані зберігаються між перезапусками сервера
  • DrinkForm — reusable компонент (використовується в create і edit)
  • Сторінка редагування /dashboard/drinks/[id]/edit працює
  • Кнопка «Редагувати» переходить на сторінку редагування
  • Код адаптовано під тему свого варіанту

Бонусні завдання

Ці завдання не є обов’язковими і не впливають на наступні практичні роботи.

  • Додати пошук та фільтр в UI списку напоїв: поле пошуку та dropdown категорій, які передають ?search= та ?category= до API
  • Додати поля emoji та available до форми DrinkForm (текстове поле або emoji picker, checkbox для наявності)
  • Додати пагінацію до GET /api/drinks: ?page=1&limit=5 з meta-інформацією (total, pages, currentPage)
  • Додати сортування через query-параметри: ?sort=price&order=asc
  • Створити Mongoose модель для другої сутності вашого проекту (наприклад, категорії, замовлення) зі зв’язком через ObjectId
  • Додати кастомний валідатор до моделі (наприклад, ціна кратна 5, назва з великої літери)
