Server Components vs Client Components

Мета заняття

На цьому практичному занятті ви:

  • Проаналізуєте існуючі компоненти та визначите їх тип (Server/Client)
  • Створите нові Client Components з інтерактивністю
  • Реалізуєте React Context для спільного стану
  • Навчитесь правильній композиції Server та Client компонентів
Передумова: Ви маєте мати працюючий Next.js проект з тижня 3 (маршрути, layouts, dashboard, динамічні сторінки). Ми продовжуємо працювати з тим самим проектом.
Приклад викладача: Кав'ярня «Кофеїн». Ви маєте адаптувати код під свою тему з 15 варіантів.

1. Аналіз існуючих компонентів

Перш ніж створювати нові компоненти, проаналізуємо те, що вже маємо. В Next.js всі компоненти за замовчуванням — Server Components. Директива 'use client' потрібна тільки для інтерактивності.

Завдання: Класифікувати існуючі компоненти

Подивіться на свої компоненти та заповніть таблицю:

Компонент Тип Причина
Header.jsx Client usePathname для активних лінків
Footer.jsx Server Статичний контент
MenuCard.jsx Server Статичний рендер (props → JSX)
MenuFilter.jsx Client useState для пошуку та фільтрації
DashboardNav.jsx Client usePathname для активних лінків
Правило: Якщо компонент не використовує hooks, events або Browser APIs — він має бути Server Component (без 'use client'). Менше JS = швидше завантаження.

Завдання: Перевірити директиви ‘use client’

Переконайтеся, що 'use client' стоїть тільки там, де дійсно потрібно:

  • Header.jsx — має 'use client' (usePathname) ✅
  • Footer.jsx — не має 'use client' ✅
  • MenuCard.jsx — не має 'use client' ✅
  • MenuFilter.jsx — має 'use client' (useState) ✅
  • DashboardNav.jsx — має 'use client' (usePathname) ✅

Завдання: Демонстрація console.log

Додайте console.log в різні компоненти і подивіться, де він з'являється:

app/page.js — Server Component
export default function Home() {
  console.log('HOME: це Server Component!')
  // Цей лог з'явиться в ТЕРМІНАЛІ (не в браузері!)

  return <div>...</div>
}
components/Header.jsx — Client Component
'use client'
export default function Header() {
  console.log('HEADER: це Client Component!')
  // Цей лог з'явиться в КОНСОЛІ БРАУЗЕРА (F12)

  return <header>...</header>
}
Важливо: Після перевірки видаліть ці console.log — вони були лише для демонстрації.
✅ Контрольна точка
  • Ви знаєте тип кожного існуючого компонента
  • ‘use client’ стоїть тільки на інтерактивних компонентах
  • console.log в Server Component з'являється в терміналі

2. Створення нових Client Components

Завдання: Створити FavoriteButton

Створіть файл components/FavoriteButton.jsx — кнопка «серце» для додавання в обране:

components/FavoriteButton.jsx
'use client'
import { useFavorites } from '@/contexts/FavoritesContext'

export default function FavoriteButton({ drinkId }) {
  const { toggleFavorite, isFavorite } = useFavorites()
  const liked = isFavorite(drinkId)

  return (
    <button
      onClick={() => toggleFavorite(drinkId)}
      className="text-xl transition hover:scale-110 cursor-pointer"
      title={liked ? 'Видалити з обраного' : 'Додати до обраного'}
    >
      {liked ? '❤️' : '🤍'}
    </button>
  )
}
Примітка: Цей компонент використовує Context (useFavorites), який ми створимо у частині 3. Поки що він не працюватиме — це нормально.

Завдання: Вбудувати FavoriteButton в MenuCard

Оновіть components/MenuCard.jsx — додайте FavoriteButton як Client «островок»:

components/MenuCard.jsx — оновлений
// Server Component з Client "островком"
import Link from "next/link";
import FavoriteButton from "./FavoriteButton";

export default function MenuCard({ id, name, description, price, emoji, category, available = true }) {
  return (
    <div className={`bg-white rounded-lg shadow-md overflow-hidden hover:shadow-lg transition ${!available ? 'opacity-60' : ''}`}>
      <div className="h-32 bg-amber-100 flex items-center justify-center">
        <span className="text-5xl">{emoji}</span>
      </div>
      <div className="p-4">
        <div className="flex justify-between items-start mb-2">
          <h3 className="text-lg font-semibold text-gray-900">{name}</h3>
          <div className="flex items-center gap-2">
            {id && <FavoriteButton drinkId={id} />}
            {available ? (
              <span className="text-xs px-2 py-1 rounded bg-green-100 text-green-700">В наявності</span>
            ) : (
              <span className="text-xs px-2 py-1 rounded bg-red-100 text-red-700">Немає</span>
            )}
          </div>
        </div>
        <p className="text-gray-600 text-sm mb-3">{description}</p>
        <div className="flex justify-between items-center">
          <span className="text-amber-700 font-bold text-lg">{price} грн</span>
          <div className="flex items-center gap-2">
            <span className="text-xs text-gray-400 bg-gray-100 px-2 py-1 rounded">{category}</span>
            {id && (
              <Link href={`/menu/${id}`} className="text-xs text-amber-700 hover:text-amber-900 font-medium">
                Детальніше &rarr;
              </Link>
            )}
          </div>
        </div>
      </div>
    </div>
  );
}
Зверніть увагу: MenuCard залишається Server Component (без 'use client'). FavoriteButton — це Client «островок» всередині Server Component. Більшість контенту картки рендериться на сервері, і лише кнопка серця потрапляє в JS bundle.

Завдання: Створити StatsCard та lib/helpers.js

Створіть Server Component для відображення статистики та допоміжну функцію:

lib/helpers.js
import { drinks } from './drinks'

export function getDrinkStats() {
  const total = drinks.length
  const available = drinks.filter(d => d.available).length
  const unavailable = total - available
  const categories = [...new Set(drinks.map(d => d.category))]
  const avgPrice = Math.round(drinks.reduce((sum, d) => sum + d.price, 0) / total)

  return { total, available, unavailable, categoriesCount: categories.length, avgPrice }
}
components/StatsCard.jsx — Server Component
// Без 'use client' — це Server Component!
const colorClasses = {
  amber: 'text-amber-700',
  green: 'text-green-600',
  blue: 'text-blue-600',
  red: 'text-red-600',
}

export default function StatsCard({ title, value, color = 'amber' }) {
  return (
    <div className="bg-white p-6 rounded-lg shadow">
      <h3 className="text-gray-500 text-sm font-bold">{title}</h3>
      <p className={`text-4xl font-bold mt-2 ${colorClasses[color] || colorClasses.amber}`}>
        {value}
      </p>
    </div>
  )
}

Завдання: Створити DrinkActions

Замініть прості кнопки «Редагувати»/«Видалити» в dashboard на Client Component з підтвердженням:

components/DrinkActions.jsx
'use client'
import { useState } from 'react'
import { useRouter } from 'next/navigation'

export default function DrinkActions({ drinkId }) {
  const [showConfirm, setShowConfirm] = useState(false)
  const router = useRouter()

  const handleDelete = () => {
    console.log(`Видалення напою ${drinkId}`)
    setShowConfirm(false)
    router.push('/dashboard/drinks')
  }

  if (showConfirm) {
    return (
      <div className="space-x-2">
        <span className="text-red-600 font-semibold mr-2">Видалити?</span>
        <button onClick={handleDelete}
          className="bg-red-600 text-white px-4 py-2 rounded hover:bg-red-700 cursor-pointer">
          Так
        </button>
        <button onClick={() => setShowConfirm(false)}
          className="bg-gray-300 text-gray-700 px-4 py-2 rounded hover:bg-gray-400 cursor-pointer">
          Ні
        </button>
      </div>
    )
  }

  return (
    <div className="space-x-2">
      <button className="bg-amber-700 text-white px-4 py-2 rounded hover:bg-amber-800 cursor-pointer">
        Редагувати
      </button>
      <button onClick={() => setShowConfirm(true)}
        className="bg-red-600 text-white px-4 py-2 rounded hover:bg-red-700 cursor-pointer">
        Видалити
      </button>
    </div>
  )
}

3. Композиція та Context

Завдання: Створити FavoritesContext

Створіть папку contexts/ та файл contexts/FavoritesContext.jsx:

contexts/FavoritesContext.jsx
'use client'
import { createContext, useContext, useState } from 'react'

const FavoritesContext = createContext()

export function FavoritesProvider({ children }) {
  const [favorites, setFavorites] = useState([])

  const toggleFavorite = (id) => {
    setFavorites(prev =>
      prev.includes(id)
        ? prev.filter(fId => fId !== id)
        : [...prev, id]
    )
  }

  const isFavorite = (id) => favorites.includes(id)

  return (
    <FavoritesContext.Provider value={{ favorites, toggleFavorite, isFavorite }}>
      {children}
    </FavoritesContext.Provider>
  )
}

export function useFavorites() {
  const context = useContext(FavoritesContext)
  if (!context) {
    throw new Error('useFavorites must be used within a FavoritesProvider')
  }
  return context
}

Завдання: Обгорнути layout FavoritesProvider

Оновіть app/layout.js — додайте FavoritesProvider навколо children:

app/layout.js — оновлений
import { FavoritesProvider } from "@/contexts/FavoritesContext";
// ... інші імпорти

export default function RootLayout({ children }) {
  return (
    <html lang="uk">
      <body className="...">
        <FavoritesProvider>
          <Header />
          <main className="flex-1">{children}</main>
          <Footer />
        </FavoritesProvider>
      </body>
    </html>
  );
}
Паттерн композиції: FavoritesProvider — це Client Component, але children (сторінки) залишаються Server Components! Це працює завдяки паттерну children — Server Components передаються через props, а не імпортуються в Client.

Завдання: Створити DashboardShell з toggle сайдбару

Створіть компоненти для згортання/розгортання сайдбару dashboard:

components/SidebarToggle.jsx
'use client'

export default function SidebarToggle({ isOpen, onToggle }) {
  return (
    <button
      onClick={onToggle}
      className="w-full text-center text-gray-400 hover:text-white transition mb-4 cursor-pointer"
      title={isOpen ? 'Згорнути меню' : 'Розгорнути меню'}
    >
      {isOpen ? '◀ Згорнути' : '▶ Меню'}
    </button>
  )
}
components/DashboardShell.jsx
'use client'
import { useState } from 'react'
import DashboardNav from './DashboardNav'
import SidebarToggle from './SidebarToggle'

export default function DashboardShell({ children }) {
  const [sidebarOpen, setSidebarOpen] = useState(true)

  return (
    <div className="flex min-h-[calc(100vh-130px)]">
      <aside className={`bg-gray-800 text-white transition-all ${sidebarOpen ? 'w-64 p-6' : 'w-16 p-2'}`}>
        <SidebarToggle isOpen={sidebarOpen} onToggle={() => setSidebarOpen(!sidebarOpen)} />
        {sidebarOpen && (
          <>
            <h2 className="text-xl font-bold mb-6">Dashboard</h2>
            <DashboardNav />
          </>
        )}
      </aside>
      <div className="flex-1 bg-gray-100 p-8">
        {children}
      </div>
    </div>
  )
}

Оновіть app/dashboard/layout.js:

app/dashboard/layout.js — оновлений
import DashboardShell from "@/components/DashboardShell";

export default function DashboardLayout({ children }) {
  return <DashboardShell>{children}</DashboardShell>;
}
Ще один приклад children: DashboardShell — Client Component (useState), але children (dashboard pages) — Server Components. Dashboard layout передає сторінки через children, не імпортуючи їх в Client.
✅ Контрольна точка
  • FavoriteButton працює — клік на серце змінює його колір
  • Стан обраного зберігається при переході між /menu та /menu/[id]
  • Dashboard сайдбар згортається та розгортається
  • Всі сторінки продовжують працювати коректно
Поширені помилки
  • useFavorites must be used within a FavoritesProvider
    FavoritesProvider не додано в layout.js або додано не на тому рівні

    Забули 'use client' на початку файлу

Підсумок

Критерії успішного виконання

  • Існуючі компоненти правильно класифіковані (Server/Client)
  • ‘use client’ стоїть тільки на інтерактивних компонентах
  • FavoriteButton працює на картках та сторінці деталей
  • FavoritesContext зберігає стан між навігацією
  • StatsCard — Server Component (без ‘use client’)
  • Dashboard показує реальну статистику з getDrinkStats()
  • DrinkActions працює з діалогом підтвердження
  • DashboardShell з toggle сайдбару
  • Паттерн children використовується в layout та DashboardShell

Бонусні завдання

Ці завдання не є обов'язковими і не впливають на наступні практичні роботи.

  • Додати Suspense з fallback для повільних компонентів
  • Реалізувати optimistic UI — миттєве оновлення UI до відповіді сервера
  • Створити файл COMPONENTS.md з таблицею всіх компонентів та їх типів
  • Порівняти bundle size через DevTools Network tab
