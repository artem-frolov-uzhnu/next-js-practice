1. Виділення компонентів Header та Footer

На минулому занятті ви писали Header (навігацію) та Footer прямо в кожній сторінці. Зараз ми виділимо їх в окремі компоненти і підключимо через layout.js, щоб вони з'являлися на всіх сторінках автоматично.

Завдання: Створити папку components

Створіть папку components в корені проекту (поруч з папкою app):

Структура проекту
my-project/
├── app/
│   ├── layout.js
│   ├── page.js
│   └── about/
├── components/      ← нова папка
│   ├── Header.jsx
│   └── Footer.jsx
└── package.json

Завдання: Підключити компоненти в layout.js

Відкрийте app/layout.js та додайте Header і Footer:

app/layout.js — оновлений
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";
import Header from "@/components/Header";
import Footer from "@/components/Footer";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",

  title: "Кав'ярня «Кофеїн»",

};

export default function RootLayout({ children }) {
  return (
    <html lang="uk">

Що таке @/components? Символ @ — це аліас для кореня проекту. Next.js автоматично налаштовує його при створенні проекту. Це зручніше, ніж писати ../../components.

Завдання: Очистити сторінки від дублікатів

Тепер Header та Footer відображаються з layout.js. Видаліть навігацію та футер з app/page.js та app/about/page.js.

Також видаліть обгортку <div className="min-h-screen"> — тепер min-h-screen забезпечує layout.

✅ Контрольна точка
  • Header з навігацією відображається на всіх сторінках
  • Footer відображається внизу кожної сторінки
  • Навігація між сторінками працює
  • В page.js та about/page.js немає дубльованої навігації
Якщо Header/Footer не з'являються
  • Перевірте імпорт: import Header from "@/components/Header"
  • Переконайтеся, що файли мають розширення .jsx
  • Назви компонентів — з великої літери: Header, Footer
  • Перевірте, що export default є в кожному компоненті

2. Компонент з props — картка товару

На минулому тижні ви створювали картки копіюючи HTML-код для кожної. Зараз ми створимо один компонент, що приймає дані через props і може відображати будь-який товар.

Завдання: Створити компонент MenuCard

Створіть файл components/MenuCard.jsx:

components/MenuCard.jsx
// Компонент картки товару з props
export default function MenuCard({ name, description, price, emoji, category, available = true }) {
  return (
    <div className={`bg-white rounded-lg shadow-md overflow-hidden hover:shadow-lg transition ${!available ? 'opacity-60' : ''}`}>
      <div className="h-32 bg-amber-100 flex items-center justify-center">
        <span className="text-5xl">{emoji}</span>
      </div>
      <div className="p-4">
        <div className="flex justify-between items-start mb-2">
          <h3 className="text-lg font-semibold text-gray-900">{name}</h3>
          {available ? (
            <span className="text-xs px-2 py-1 rounded bg-green-100 text-green-700">
              В наявності
            </span>
          ) : (
            <span className="text-xs px-2 py-1 rounded bg-red-100 text-red-700">
              Немає
            </span>
          )}
        </div>
        <p className="text-gray-600 text-sm mb-3">{description}</p>
        <div className="flex justify-between items-center">
          <span className="text-amber-700 font-bold text-lg">{price} грн</span>
          <span className="text-xs text-gray-400 bg-gray-100 px-2 py-1 rounded">{category}</span>
        </div>
      </div>
    </div>
  );
}
Зверніть увагу:
  • Деструктуризація props — { name, description, price, ... }
  • Значення за замовчуванням — available = true
  • Умовний рендеринг — тернарний оператор для статусу доступності
  • Умовні класи — !available ? 'opacity-60' : ''

  • Кожна картка має унікальний key

3. Інтерактивність зі useState — фільтрація меню

Тепер створимо нову сторінку /menu з повним меню, пошуком та фільтрацією по категорії. Для цього потрібен хук useState і директива 'use client'.

Завдання: Створити компонент MenuFilter

Створіть файл components/MenuFilter.jsx:

components/MenuFilter.jsx
'use client'
import { useState } from 'react'
import MenuCard from './MenuCard'

// Дані меню (адаптуйте під свою тему!)
const menuItems = [
  { id: 1, name: "Еспресо", description: "Класичний еспресо.", price: 45, emoji: "☕", category: "Кава", available: true },
  { id: 2, name: "Капучино", description: "Еспресо з пінкою.", price: 65, emoji: "🥛", category: "Кава", available: true },
  { id: 3, name: "Латте", description: "Кава з молоком.", price: 70, emoji: "🍵", category: "Кава", available: true },
  { id: 4, name: "Американо", description: "Еспресо з водою.", price: 50, emoji: "☕", category: "Кава", available: true },
  { id: 5, name: "Раф", description: "Вершковий кавовий напій.", price: 75, emoji: "🍨", category: "Кава", available: false },
  { id: 6, name: "Круасан", description: "Свіжий французький круасан.", price: 55, emoji: "🥐", category: "Їжа", available: true },
  { id: 7, name: "Чізкейк", description: "Чізкейк з ягідним соусом.", price: 95, emoji: "🍰", category: "Їжа", available: true },
  { id: 8, name: "Зелений чай", description: "Японський зелений чай.", price: 55, emoji: "🍃", category: "Чай", available: true },
]

// Категорії для фільтрів
const categories = ["Всі", ...new Set(menuItems.map(item => item.category))]

export default function MenuFilter() {
  // Стан для пошуку та фільтрації
  const [search, setSearch] = useState('')
  const [activeCategory, setActiveCategory] = useState('Всі')
  const [showAvailableOnly, setShowAvailableOnly] = useState(false)

  // Логіка фільтрації
  const filteredItems = menuItems.filter(item => {
    const matchesSearch = item.name.toLowerCase().includes(search.toLowerCase())
    const matchesCategory = activeCategory === 'Всі' || item.category === activeCategory
    const matchesAvailability = !showAvailableOnly || item.available
    return matchesSearch && matchesCategory && matchesAvailability
  })

  return (
    <div>
      {/* Пошук */}
      <input
        type="text"
        value={search}
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Пошук..."
        className="w-full px-4 py-3 border rounded-lg mb-6 focus:outline-none focus:border-amber-500 text-gray-900"
      />

      {/* Кнопки категорій */}
      <div className="flex flex-wrap gap-2 mb-4">
        {categories.map(cat => (
          <button
            key={cat}
            onClick={() => setActiveCategory(cat)}
            className={`px-4 py-2 rounded-lg font-medium transition cursor-pointer ${
              activeCategory === cat
                ? 'bg-amber-700 text-white'
                : 'bg-gray-200 text-gray-700 hover:bg-gray-300'
            }`}
          >
            {cat}
          </button>
        ))}
      </div>

      {/* Чекбокс */}
      <label className="flex items-center gap-2 mb-6 cursor-pointer">
        <input
          type="checkbox"
          checked={showAvailableOnly}
          onChange={(e) => setShowAvailableOnly(e.target.checked)}
          className="w-4 h-4"
        />
        <span className="text-gray-700">Тільки в наявності</span>
      </label>

      {/* Кількість */}
      <p className="text-sm text-gray-500 mb-4">
        Знайдено: {filteredItems.length} з {menuItems.length}
      </p>

      {/* Картки або повідомлення */}
      {filteredItems.length > 0 ? (
        <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
          {filteredItems.map(item => (
            <MenuCard key={item.id} {...item} />
          ))}
        </div>
      ) : (
        <p className="text-center py-12 text-gray-400">Нічого не знайдено</p>
      )}
    </div>
  )
}
Зверніть увагу: На першому рядку обов'язково стоїть 'use client' — без нього useState не працюватиме! В Next.js компоненти за замовчуванням виконуються на сервері.

Завдання: Створити сторінку /menu

Створіть файл app/menu/page.js:

app/menu/page.js
import MenuFilter from "@/components/MenuFilter";

export default function MenuPage() {
  return (
    <div>
      <section className="bg-gradient-to-r from-amber-800 to-amber-600 text-white py-12">
        <div className="container mx-auto px-4 text-center">
          <h1 className="text-4xl font-bold mb-2">Наше меню</h1>
          <p className="text-lg opacity-90">Знайдіть свій ідеальний напій або перекус</p>

        <div className="container mx-auto px-4">

        </div>

Чому page.js не має 'use client'? Сторінка menu/page.js — це Server Component. Вона лише підключає клієнтський компонент MenuFilter. Інтерактивність «живе» в окремому компоненті — це рекомендований паттерн в Next.js.

Мінімальні вимоги:

  • Не менше 8 позицій у масиві
  • Мінімум 3 категорії
  • Хоча б 1-2 позиції з available: false
✅ Контрольна точка
  • Сторінка /menu відкривається
  • Пошук фільтрує картки в реальному часі
  • Кнопки категорій працюють (активна кнопка підсвічується)
  • Чекбокс "Тільки в наявності" працює
  • Відображається кількість знайдених позицій
  • При відсутності результатів показується повідомлення
Поширені помилки
  • Error: useState is not a function
    Забули 'use client' на першому рядку файлу
  • Each child in a list should have a unique "key"
    Додайте key={item.id} в .map()
  • Фільтрація не працює
    Перевірте, що value={search} та onChange присутні на input

Підсумок

Критерії успішного виконання

  • Header та Footer виділені в компоненти
  • Header/Footer підключені в layout.js
  • MenuCard приймає та відображає props
  • Головна сторінка використовує MenuCard через .map()
  • Створена сторінка /menu з пошуком
  • Фільтрація по категорії працює
  • Чекбокс "Тільки в наявності" працює
  • Дані адаптовані під обрану тему
